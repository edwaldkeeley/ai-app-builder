Fix live streaming: backend now extracts clean message text from JSON

Root cause: The backend was sending raw JSON tokens (including JSON key prefixes
and suffixes) as message_chunk events. The frontend's stripJsonWrapper tried to
clean this up but produced garbled output on partial tokens.

Fix:
1. Backend (ai_service.py): Now uses regex to extract ONLY the message text value
   from the streaming JSON. Two strategies:
   a) Primary: message_re matches the full message value before files key
   b) Fallback: simpler regex matches partial message value during early tokens
   Only clean text is sent as message_chunk - no raw JSON reaches the frontend.
2. Frontend (useChat.ts): Removed stripJsonWrapper entirely. The backend now
   sends clean text, so the frontend just appends deltas directly.

The UI now shows clean character-by-character streaming of the AI response.
